# Critical Pattern Recognition

## Pattern 1: Content Rendering Under Sidebar

**Visual Symptom:** Main content appears beneath the sidebar instead of beside it

**Root Cause Analysis:**
1. Sidebar not taking space in document flow (usually fixed positioning without spacer)
2. Invalid CSS custom property syntax causing width to be 0
3. Missing `min-w-0` on flex containers

**Systematic Fix:**
```tsx
// ❌ WRONG - Content will render under sidebar
<div className="relative flex min-h-screen w-full">
  <Sidebar className="w-(--sidebar-width)" /> {/* Invalid syntax */}
  <div className="flex-1">{children}</div>
</div>

// ✅ CORRECT - Proper flex layout
<div className="flex min-h-screen w-full"> {/* No 'relative' */}
  <Sidebar className="w-[var(--sidebar-width)]" /> {/* Valid syntax */}
  <div className="flex min-w-0 flex-1 flex-col">{children}</div> {/* min-w-0 is CRITICAL */}
</div>
```

## Pattern 2: Sidebar Collapse Not Hiding Text

**Visual Symptom:** Text labels still visible when sidebar is collapsed to icons

**Root Cause Analysis:**
1. Data attribute classes not being generated by Tailwind
2. Missing proper group utilities
3. Incorrect state management

**Systematic Fix:**
```tsx
// Strategy 1: CSS Classes (requires Tailwind config)
<span className="group-data-[collapsible=icon]:hidden">Label</span>

// Strategy 2: Conditional Rendering (always works)
{state !== "collapsed" && <span>{label}</span>}

// Strategy 3: Both for maximum compatibility
<span className={cn(
  "truncate",
  "group-data-[collapsible=icon]:hidden",
  state === "collapsed" && "hidden"
)}>
  {label}
</span>
```
